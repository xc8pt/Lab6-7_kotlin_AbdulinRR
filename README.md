# Лабораторная работа №6-7. Взаимодействие с пользовательским интерфейсом и State в Jetpack Compose. Написание автоматических тестов
## Цель работы
Познакомиться с понятием состояния (state) в Android-приложениях и научиться
использовать его в интерфейсе с помощью Jetpack Compose. В результате
лабораторной работы будет создано приложение калькулятор чаевых (Tip Time),
которое автоматически рассчитывает сумму чаевых. Изучить автоматизированные
тесты в Android.
---
В данной работе мы начуились: 
 - Создавать функции для расчёта чаевых
 - Подключение строки (strings.xml)
Так же изучили:
 - Композиция
   Compose выполняет компонуемые функции и формирует интерфейс на основе
   текущих данных. Интерфейс описывается декларативно — через состояние, а не
   пошаговые изменения.
 - Рекомпозиция
   При изменении State Compose автоматически пересоздаёт нужные части
   интерфейса.
 - Отслеживание изменений
   Compose запоминает используемые значения State и запускает рекомпозицию при
   их изменении. Единственный способ обновить интерфейс — изменить State.
 - Использование MutableState
   Обычные переменные не отслеживаются Compose.
   mutableStateOf() создаёт наблюдаемое и изменяемое состояние.
## Что умеет наше приложение?
Наше приложение может расчитывать сумму чаевых для официантов.
Введённое значение не исчезает, а корректно сохраняется и отображается.
Есть также метки (label), которая подсказывает пользователю, какие данные нужно вводить.
---
Что делает метка?
 - пока поле не активно — текст метки находится внутри поля;
 - при нажатии — метка переносится вверх.
